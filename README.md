Информационная система для интернет-магазина (SQL Server)



&#x20;Описание проекта

Проект представляет собой разработку и оптимизацию базы данных для информационной системы интернет-магазина.  

Система поддерживает управление каталогом товаров, корзиной покупок, заказами и клиентами, а также включает аналитические и отчетные возможности.



Проект выполнен на \*\*Microsoft SQL Server\*\* и предназначен для демонстрации навыков проектирования, оптимизации и администрирования баз данных.



\---



&#x20;Используемая СУБД

\- Microsoft SQL Server



\---



&#x20;Структура базы данных

База данных включает 11 таблиц:

\- Customers

\- CustomerAddresses

\- Categories

\- Products

\- ProductImages

\- Carts

\- CartItems

\- Orders

\- OrderItems

\- Payments

\- Shipments



Между таблицами настроены первичные и внешние ключи для обеспечения ссылочной целостности данных.



ER-диаграмма представлена в каталоге 'diagrams'.

&#x20;Оптимизация производительности

\- Создано 5 индексов

\- Реализован 1 CTE

\- Реализован 1 подзапрос



\---



&#x20;Представления (Views)

Созданы представления для упрощения аналитических запросов и формирования отчетов.



\---



&#x20;Триггеры

Реализовано 2 триггера для автоматического реагирования на изменения данных.



\---



&#x20;Мониторинг производительности

Файл '10\_monitoring.sql' содержит SQL-запросы для получения метрик производительности SQL Server, включая:

\- активные запросы

\- блокировки

\- использование и фрагментацию индексов

\- ключевые бизнес-показатели (KPI)



\---



&#x20;Безопасность

Настроен механизм управления доступом:

\- создана роль 'report\_reader'

\- выданы права 'SELECT' на аналитические представления



\---



&#x20;Отчеты и аналитика

Файл '12\_reports.sql' содержит примеры бизнес-отчетов:

\- анализ продаж и активности клиентов

\- заказы по статусам

\- самые продаваемые товары



\---



&#x20;JSON-данные

В проекте реализована интеграция JSON-данных:

\- добавлено поле 'extra\_data' в таблицу Orders

\- используется хранение и чтение JSON через функции 'JSON\_VALUE' и 'ISJSON'



Это позволяет гибко хранить дополнительные параметры заказов без изменения структуры таблиц.
